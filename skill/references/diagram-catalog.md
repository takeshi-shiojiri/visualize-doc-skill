# 図解タイプカタログ

> **このファイルは必要時のみ参照する。** 毎回読み込まない。
> Step 2（構造分析・図解タイプ選択）で、SKILL.md本体の選択ルールでカテゴリまで絞ったあと、
> 具体的な図解タイプとレンダリングパターンを確認するために読む。

---

## カテゴリ1: 全体像・分類を見せたい

| 図解タイプ | レンダリング | 適用シーン |
|-----------|------------|-----------|
| 階層ツリー | SVGエンジン `build_tree()` | 親子関係、カテゴリ分け、機能→サブ機能 |
| マインドマップ | SVGエンジン `build_tree(direction='LR')` | 発想の広がり、連想のネットワーク |
| マトリクス(2軸/多軸) | SVGエンジン `build_matrix()` | 分類・比較（重要度×緊急度 等） |
| ベン図 | SVG直接記述 | 集合の重なり、共通点/差分 |
| オントロジー/タクソノミー | SVGエンジン `build_tree()` | 概念の定義と包含関係 |

### 選択ヒント
- 「分類したい」「整理したい」「全体像を見たい」→ このカテゴリ
- 親子関係が明確 → 階層ツリー
- 中心テーマから放射状に広がる → マインドマップ
- 2軸以上で分類 → マトリクス
- 重なりや共通部分が重要 → ベン図
- 用語や概念の包含関係 → オントロジー

---

## カテゴリ2: 因果・メカニズムを説明したい

| 図解タイプ | レンダリング | 適用シーン |
|-----------|------------|-----------|
| 因果ループ図 | SVGエンジン `build_flow()` + 循環エッジ | 強化ループ/抑制ループ（成長、負の連鎖） |
| ロジックツリー | SVGエンジン `build_tree()` | 原因分解(Why)、打ち手分解(How) |
| IPO図(入力→処理→出力) | SVGエンジン `build_flow(direction='LR')` | 仕組みの骨格を最短で伝える |
| フィードバック図 | SVGエンジン `build_flow()` + 循環エッジ | 計測→学習→改善の循環 |

### 選択ヒント
- 「なぜ」「原因は」→ ロジックツリー
- 循環する構造、連鎖 → 因果ループ図 / フィードバック図
- 入力と出力が明確 → IPO図

### レンダリング補足: 因果ループ図
- 強化ループのエッジには `label="+"`, `color=success_teal` を付与
- 抑制ループのエッジには `label="-"`, `color=alert_red` を付与
- ループ名はsubgraphのlabelで表現

---

## カテゴリ3: プロセス・手順・時間を見せたい

| 図解タイプ | レンダリング | 適用シーン |
|-----------|------------|-----------|
| フローチャート | SVGエンジン `build_flow()` | 分岐がある手順、判断点が重要な業務 |
| シーケンス図 | SVG直接記述 | 登場人物間のやりとりの時系列 |
| スイムレーン図 | SVGエンジン `build_journey_map()` | 部門/役割ごとの責務を分けて手順を示す |
| タイムライン/ロードマップ | SVGエンジン `build_timeline()` | 時系列で変化を追う |
| 状態遷移図 | SVGエンジン `build_flow()` | 状態の定義と遷移条件 |

### 選択ヒント
- 分岐(if/else)がある → フローチャート
- 複数の登場人物のやりとり → シーケンス図
- 部門横断の業務フロー → スイムレーン図
- 時間軸が重要 → タイムライン
- 状態の変化が重要 → 状態遷移図

### レンダリング補足: フローチャート
- 判断ノード: `type='decision'`（菱形）, highlight_yellow
- 処理ノード: `type='process'`（角丸矩形）, アクセントサイクル
- 開始/終了: `type='start'`/`type='end'`（楕円）, success_teal/alert_red

### レンダリング補足: シーケンス図
- 横軸: 参加者（アクター）を等間隔配置
- 縦軸: 時間（上→下）
- メッセージ: 矢印 + ラベル（実線=同期、破線=非同期）
- ライフライン: 縦の点線

---

## カテゴリ4: 関係性・構造の"配置"を見せたい

| 図解タイプ | レンダリング | 適用シーン |
|-----------|------------|-----------|
| レイヤー図 | SVGエンジン `build_layer()` | 基盤→アプリ→体験の層構造 |
| コンポーネント図 | Graphviz dot → SVG + subgraph | 部品の構成と依存関係 |
| ネットワーク図 | Graphviz neato/fdp → SVG | 多対多のつながり |
| 概念地図(マップ) | SVG直接記述 | 近さ/遠さで意味を表現 |

### 選択ヒント
- 層・レイヤーが重要 → レイヤー図
- システムやサービスの構成 → コンポーネント図
- 多対多の複雑な接続 → ネットワーク図
- ポジションや距離感が意味を持つ → 概念地図

### レンダリング補足: レイヤー図
- 下層から上層へ横長ボックスを積み上げる
- 各層の色はLEVEL_COLORSの淡色を使用
- 層間の矢印は上向き

---

## カテゴリ5: 比較・差分・意思決定を支援したい

| 図解タイプ | レンダリング | 適用シーン |
|-----------|------------|-----------|
| 比較表(スコアカード) | SVGエンジン `build_matrix()` | 選定理由を残す（重み付け可） |
| ポジショニングマップ | SVG直接記述 | 競合や選択肢の位置関係 |
| ギャップ図(As-Is/To-Be) | SVGエンジン `build_two_columns()` | 現状と理想の差分 |
| トレードオフ曲線 | SVG直接記述 | 速度vs品質、コストvs効果 |

### 選択ヒント
- 複数案の評価・比較 → 比較表
- 2軸上での相対位置 → ポジショニングマップ
- 現状と理想のギャップ → ギャップ図
- 二律背反の関係 → トレードオフ曲線

---

## カテゴリ6: 人の体験・認知を説明したい

| 図解タイプ | レンダリング | 適用シーン |
|-----------|------------|-----------|
| カスタマージャーニーマップ | SVGエンジン `build_journey_map()` | 行動×感情×課題×機会 |
| サービスブループリント | SVGエンジン `build_journey_map()` | フロント/バック/仕組みの対応関係 |
| ペルソナ→課題→価値 対応図 | SVGエンジン `build_tree()` or `build_flow()` | 誰の何をどう解くか |

### 選択ヒント
- 顧客体験の全体像 → CJM
- フロント/バックの対応関係 → サービスブループリント
- ペルソナと価値の対応 → ペルソナ→課題→価値図

### レンダリング補足: CJM
- 横軸: フェーズ（各フェーズにカプセル見出し）
- 縦軸レーン: 行動、感情、課題、施策（レーン間は破線区切り）
- 感情曲線: 折れ線グラフをオーバーレイ
- 感情ラベル: 絵文字は使わず日本語テキスト（sanitize_text適用）
- 要素数上限: フェーズ最大6、レーン最大5

### レンダリング補足: サービスブループリント
- 横軸: フェーズ
- 縦軸レーン: 顧客行動、フロントステージ、バックステージ、サポートプロセス
- レーン間に「可視線」「内部相互作用線」を破線で描画

## 要素数の上限ガイドライン

| 図解タイプ | 推奨上限 | 超過時の対応 |
|-----------|---------|-----------|
| ツリー系 | 深さ4段、幅8ノード | サマリー化を提案 |
| フローチャート | 15ステップ | 分割を提案 |
| マトリクス | 6×6 | 代表例に絞る |
| CJM/ブループリント | 6フェーズ×5レーン | フェーズ統合を提案 |
| ネットワーク図 | 20ノード | クラスタリングを提案 |
| シーケンス図 | 6アクター、15メッセージ | 分割を提案 |
